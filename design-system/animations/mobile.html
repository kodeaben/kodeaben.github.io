Animation language · Mobile

Built for thumbs.

The desktop motion language carries over — but mobile changes the rules. No hover state, finger occlusion, smaller hit targets, battery cost. Here are the same animations adapted for touch, plus four patterns that only exist on mobile.

!
The mobile shift in one sentence
Hover collapses into press, durations get ~30% shorter, the bounce gets ~40% smaller, and we add four touch-native patterns. The rest stays.

C. Performance & access
✓
GPU-only properties
All animations use only transform and opacity. Box-shadow transitions are limited to small offsets to keep them cheap. Never animate width, top, or filter on mobile.
♿
prefers-reduced-motion
All transforms collapse to opacity-only. Tab indicator stops sliding. Bottom sheet appears with a fade. Toasts skip the bounce. Press states keep the shadow snap because it's state communication, not decoration.
📳
Pair animation with haptics
Buttons: vibrate(8) on press. Success states: vibrate([10,40,10]). Errors: vibrate([20,20,20]). Always opt-in via a settings toggle.
